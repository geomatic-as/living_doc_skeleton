# decisions

ADRs and decision logs. One file per decision, named like `0001-use-postgres.md` (zero-padded sequence + slug).

---

## Template — decision record

> Copy the sections below into a new `.md` file (e.g. `0001-use-postgres.md`).

### Status
Proposed / Accepted / Superseded / Deprecated.

### Date
YYYY-MM-DD.

### Context
What problem we're solving and the constraints.

### Decision
What we're doing.

### Consequences
What changes as a result — positive and negative.

### Alternatives considered
- option — why rejected
